# v0.1 Demo Workflow

This guide shows three ways to run the PolicyLens v0.1.0-alpha workflow. All demo policy content is fictional.

## Prerequisites

Start the API and apply migrations:

```bash
cp .env.example .env
docker compose up --build
docker compose exec api alembic upgrade head
```

Open:

- Web: http://localhost:3000
- API docs: http://localhost:8000/docs

## Method 1: Web Workbench

1. Open http://localhost:3000/projects/new.
2. Create a project.
3. Open the project workbench.
4. Upload `examples/demo-policy-notice.txt` with role `policy`.
5. Parse the policy document.
6. Ingest the parsed policy as a policy library record, using `examples/demo-policy-metadata.json` for metadata.
7. Upload `examples/demo-research-article.txt` with role `research_article`.
8. Parse the research article.
9. Run Analysis.
10. Inspect the tabs for steps, plan, claims, policy matches, evidence, impact matrix, and Markdown report.
11. Create a report export bundle.
12. Create a policy original export bundle.
13. Download the export bundles from the Export tab.

## Method 2: HTTP API

Use `examples/demo-workflow.http` with VS Code REST Client or a similar tool. The file contains placeholders for IDs created during the workflow.

High-level API sequence:

```text
GET /api/health
POST /api/projects
POST /api/documents/upload
POST /api/documents/{document_id}/parse
POST /api/policies/from-document
POST /api/analysis/jobs
GET /api/analysis/jobs/{job_id}/report
POST /api/exports/report
POST /api/exports/policy-originals
```

## Method 3: Scripted Smoke Test

With the API running:

```bash
python3 scripts/e2e_demo.py
```

To point at a different API origin:

```bash
POLICYLENS_API_BASE_URL=http://localhost:8000 python3 scripts/e2e_demo.py
```

Expected success output:

```text
PolicyLens v0.1 demo workflow completed successfully.
Project ID: ...
Policy ID: ...
Analysis Job ID: ...
Report Export ID: ...
Policy Export ID: ...
```

The script does not call external LLMs, does not download ZIP files into the repository, and uses only the demo files under `examples/`.

## What This Demo Proves

- Project creation works.
- Upload storage and sha256 calculation work.
- Basic parsing writes chunks.
- Policy ingestion writes policy, version, and sections.
- Research Plan execution creates claims, policy matches, evidence, impact matrix, and Markdown report.
- Report export and policy original export create completed export records.

## What This Demo Does Not Prove

- OCR
- RAG, embeddings, or Qdrant retrieval
- external LLM analysis
- formal investment or legal conclusions
- PPT, DOCX, or PDF exports
- production authentication or permissions
